1. Sirius: Context & Concepts 

Sirius in a nutshell

  • Specification environment to (Specifier/Developer)
    • Define custom multi-view worbenches (Diagrams, Trees, Tables Editors...)
    • With little technical knowledge
    • Get a working modeler fast (POC)
    • Highly customizable (within the tool our outside - Java & extension points)

  • The associated runtime (Customer/End-user)
    • Work with data (models) in an environment adapted to his needs (Viewpoint-based)
    • Custom model structure (DSL), custom graphical representations (diagrams, tables, trees)
    • Simplified workflow (Modeling Project)

Sirius in a nutshell

  • Based on Eclipse technologies
    • EMF, EMF Transaction, GEF/GMF, SWT/JFace ???Acceleo???
  • Joint development by Thales & Obeo for the last 6 years
    1. Thales has the requirements and concrete use cases
    2. Obeo develops the tech, making it as generic as possible
      - no Thales-specific code in Sirius
    3. Thales configures/customizes the tech: create their own modelers and modeling workbench
      • in this case, a complete modeling environement with also a lot of normal Eclipse/EMF development in Java
      • everything integrates naturally in the Eclipse platform and workbench
    4. Rinse and repeat, every 3 months for 6 about years
